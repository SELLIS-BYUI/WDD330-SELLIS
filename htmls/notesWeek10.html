Week 10 reading:

This week I learned that form validation is very helpful and beneficial in a lot of ways. One is that is allows forms to require certain types of information. And to show the user this through different types of pop ups or highlighting. Some of these validations comes in shape of built-in client side and JavaScript type. With JavaScript is gives a lot more freedom and more specialized types of validation. But with HMTL, here is some built in validation, “required: Specifies whether a form field needs to be filled in before the form can be submitted. minlength and maxlength: Specifies the minimum and maximum length of textual data (strings). min and max: Specifies the minimum and maximum values of numerical input types. type: Specifies whether the data needs to be a number, an email address, or some other specific preset type. pattern: Specifies a regular expression that defines a pattern the entered data needs to follow” (MDN web docs). Code example:

(JS) const email = document.getElementById("mail"); email.addEventListener("input", function (event) { if (email.validity.typeMismatch) { email.setCustomValidity("I am expecting an e-mail address!"); } else { email.setCustomValidity(""); } });
